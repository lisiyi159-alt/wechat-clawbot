# 家庭旅行手账 · 微信小程序

一个私密的家庭旅行相册：记录各地旅行的照片与文字，方便日后翻阅，分享给指定家人。

- **微信白名单访问**：只有被加入小程序「体验成员」名单的家人才能打开，其他人根本进不来，无需邀请码
- **一键分享给家人**：转发卡片带封面图和标题，在家庭群里点一下就能发
- **图文游记**：照片 + 文字混排，按时间倒序排列
- **一键导出留存**：把某段旅行的照片保存到手机相册、文字复制到剪贴板，长期备份
- **定期提醒**：每三个月弹窗提醒你导出备份（数据虽存在云端，备份更安心）
- **零服务器**：基于微信云开发（数据库 + 云存储 + 云函数）

## 目录结构

```

  app.js / app.json / app.wxss

```

## 接入与运行步骤

1. **导入项目**：用「微信开发者工具」导入本目录，AppID 填你自己的小程序 AppID
   （把 `project.config.json` 里的 `REPLACE_WITH_YOUR_APPID` 换成你的，或在导入时填写）。

2. **开通云开发**：开发者工具顶部点「云开发」，新建一个环境，复制环境 ID，
   填入 `miniprogram/config.js` 的 `ENV_ID`。

3. **创建数据库集合**：在云开发控制台「数据库」中新建两个集合：
   - `members`（成员/角色）
   - `trips`（旅行记录）

   权限都可设为「仅创建者可读写」即可（所有读写都走云函数，已在服务端做权限校验）。

4. **上传云函数**：分别右键 `cloudfunctions/login`、`trips`，选择
   「上传并部署：云端安装依赖」。

5. **配置微信白名单（关键）**：登录 [微信公众平台](https://mp.weixin.qq.com)
   → 「管理」→「成员管理」，把要给看的家人微信加为**体验成员**。
   小程序保持「体验版」状态、不发布到线上，则只有名单内的家人能打开。

6. **首次进入**：你自己第一个打开小程序，会被自动设为管理员（admin），
   右下角出现 ＋ 可以发布旅行。家人通过你转发的卡片或体验版二维码打开，
   自动登记为成员，可查看 / 导出 / 再分享。

## 权限模型

| 角色 | 谁 | 能力 |
| --- | --- | --- |
| admin | 第一个打开小程序的人（你） | 发布 / 编辑 / 删除 / 查看 / 导出 / 分享 |
| member | 微信白名单内的其他家人 | 查看 / 导出 / 分享 |

> 访问权限由微信「体验成员」名单控制：没被加入名单的人打不开小程序，
> 因此程序内不再需要邀请码。想让家人也能发布游记？编辑
> `cloudfunctions/trips/index.js` 中 `action === 'save'` 的判断，
> 把 `role !== 'admin'` 改为 `false` 后重新部署即可。

## 关于数据保存

云开发数据库与云存储中的数据**不会按天自动删除**，可长期保存。注意：

- 免费体验环境主要面向开发阶段，**小程序正式上线后第 15 天到期**，需转为付费续费
  （转付费时环境 ID 和数据都不变，不会丢失）。体验版长期使用同样建议尽早开通付费。
- 无论用什么平台，都建议用本程序的「导出留存」**定期把照片和文字备份到本地**。
  程序会每三个月提醒你一次。
